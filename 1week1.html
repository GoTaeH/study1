UEFI

: UEFI(Unified Extensible Firmware Interface : 통합 확장 펌웨어 인터페이스)는 기존 바이오스(Legacy BIOS : Legacy Basic Input/Output System)를 대체하는 시스템으로, 운영체제와 플랫폼 펌웨어 간의 소프트웨어 인터페이스를 정의하는 공개 사양이다.

UEFI의 특징 몇 가지
    1. GUI(Graphical User Interface) 제공
    2. 마우스 커서 활성화
    3. 4개 이상의 주 파티션 지원하는 GPT(GUID Partition Table) 존재 [기존 바이오스 : MBR]
    4. 보안 부팅(Secure Boot) 활성화
    5. 기존 바이오스보다 더 빠른 부팅 속도
    6. 2.2TB 이상의 대용량 디스크 인식 가능

메인보드의 BIOS 모드 확인 방법

Windows + R키를 눌러 msinfo32을 입력->확인, 이를 통해 시스템 정보 화면의 BIOS 모드에서 UEFI인지 레거시인지를 확인할 수 있다.

PXE

: PXE(Preboot Execution Environment)는 컴퓨터 네트워크 부팅을 위한 표준화된 환경을 제공하는 프로토콜 및 기술이다. PXE를 사용하면 로컬 디스크가 아닌 네트워크를 통해 운영체제나 다른 네트워크 리소스를 부팅할 수 있으며, 대규모 시스템 관리와 유지 보수 작업에 유용하게 활용된다.

PXE 주요 구성 요소
  • PXE 클라이언트 : 네트워크 인터페이스 카드에 내장된 PXE Boot ROM을 사용하여 부팅 과정을 시작
  • DHCP 서버 : 네트워크에 IP주소와 기타 네트워크 구성 정보를 할당
  • TFTP 서버 : 부트로더 및 커널 파일 전송

PXE 부팅 과정
    1. 컴퓨터를 켜면 PXE 클라이언트는 DHCP 서버로부터 IP주소 요청
    2. DHCP 서버는 PXE 클라이언트에게 IP주소와 PXE 부팅을 위한 추가적인 DHCP 옵션 정보 제공
    3. PXE 클라이언트는 TFTP 서버에서 부트 이미지 파일 다운
    4. 다운로드된 부트 이미지 실행해 운영체제 또는 다른 네트워크 리소스 부팅

라즈베리파이

: 라즈베리파이(Raspberry Pi)는 영국의 라즈베리 파이 재단에서 개발한 싱글보드 컴퓨터이다. 라즈베리파이는 신용카드 크기 정도의 소형 싱글보드 컴퓨터이기 때문에 휴대성이 좋고, 작은 공간에 설치가 가능하다. 또한, 저렴한 가격으로 구매가 가능해 교육 및 개인 프로젝트에 활용되고 있다. 모델이 다양하며, 다양한 운영체제를 지원하는데 가장 많이 사용되는 운영체제는 Raspbian(라즈베리파이용으로 개발된 Debian 기반의 운영체제)이다. GPIO 포트를 통해 센서, 모터, LED 등의 외부 장치와 연결도 가능하며, 다양한 연결 옵션되 제공되어 다양한 용도로 활용된다. 특히 IoT 프로젝트 등을 학습하고 구현하는 데 많은 사람들이 사용하고 있다.

VPN

: VPN(Virtual Private Network)는 가상 사설망을 의미한다. 이는 인터넷을 통해 디바이스 간에 사설 네트워크 연결을 생성한다. 퍼블릭 네트워크를 통해 데이터를 안전하게 익명으로 전송하는 데 사용하며, 사용자 IP주소를 마스킹하고 데이터를 암호화하여 수신 권한이 없는 사람이 읽을 수 없도록 한다.

VPN 주요 기능
  • 개인정보 보호 : 사용자의 IP주소를 숨기고, 사용자의 실제 위치를 숨길 수 있다. 이를 통해 개인정보를 보호하고, 익명성 유지가 가능하다.
  • 보안 : 데이터를 암호화하여 제 3자로부터의 무단 접근을 방지한다. 이는 보안이 약한 곳에서 사용자의 개인정보를 보호하는 데 도움이 된다.
  • 지역 제한 우회 : 지역 제한된 콘텐츠에 접근할 수 있다. VPN으로 서버 위치를 변경하면 지리적으로 제한된 콘텐츠에 액세스할 수 있다.

포트포워딩

: 포트포워딩(Port Forwarding)은 인터넷에서 네트워크 장치나 서비스에 접속하기 위해 사용되는 기술로 외부 아이피 포트번호와 내부 아이피 포트번호를 연결해주는 기능이다. 즉, 특정 서비스나 애플리케이션이 네트워크 내부에서 외부로의 연결을 필요로 할 때, 포트 포워딩을 사용하여 외부로부터의 요청을 내부 서버나 장치로 전달 할 수 있다.

포트포워딩 종류
  • 로컬 포트포워딩 : 클라이언트 컴퓨터에서 시작되어 클라이언트의 로컬 포트를 통해 원격 서버로의 연결을 전달한다. 이를 통해 클라이언트와 원격 서버 간의 직접적인 통신이 가능하다.
  • 원격 포트포워딩 : 원격 서버에서 시작되어 원격 포트를 통해 클라이언트로의 연결을 전달한다. 원격 서버가 클라이언트에서 서비스를 제공하도록 허용한다.
  • 동적 포트포워딩 : SSH를 통해 인터넷을 안전하게 탈출하여 프록시 서버로 사용한다. 이를 통해 클라이언트 컴퓨터의 모든 인터넷 트래픽을 SSH 연결을 통해 보안적으로 전달할 수 있다.

타입스크립트와 자바스크립트

자바스크립트

: 객체 기반의 스크립트 프로그래밍 언어이다. 자바스크립트는 웹의 동작을 구현할 수 있다. HTML, CSS와 함께 웹을 구성하는 요소 중 하나로, 주로 웹 브라우저 내에서 사용되며, 서버 프로그래밍에도 사용된다. 또한, 자바스크립트는 동적이며, 타입을 명시할 필요가 없는 인터프리터 언어이고, 객제 지향형/함수형 프로그래밍을 모두 표현할 수 있다.


타입스크립트

: Microsoft에서 개발한 자바스크립트를 기반으로 정적 타입 문법을 추가한 프로그래밍 언어이다. 타입스크립트는 ES6(ECMAScript 6)에서 새롭게 사용된 문법을 포함하고 있으며, 객체 지향 프로그래밍 패턴(class, interface, extends 등)을 제공한다. 또, 높은 수준의 코드 탐색과 디버깅을 할 수 있으며, 자바스크립트와 완전히 호환된다. 널리 사용되는 언어이므로 다양한 라이브러리와 프레임워크(Angular, React, Vue.js 등)의 지원을 받을 수 있다.


타입스크립트와 자바스크립트의 비교 (타입스크립트를 중점으로)

자바스크립트는 동적 타입의 인터프리터 언어로 런타임에서 오류를 발견할 수 있지만, 타입스크립트는 정적 타입의 컴파일 언어로 코드 작성 단계에서 오류를 확인할 수 있고, 미리 타입을 결정하기 때문에 실행속도가 매우 빠르다.
타입스크립트는 정적 타입 언어이므로 컴파일 시간은 조금 걸리지만 안정성이 보장되어 대규모 프로젝트에서 사용하기 유리하며, 자바스크립트는 동적 타입 언어이므로 런타임 속도는 빠르지만 타입 안정성이 보장되지 않아 빠른 프로토타이핑 및 작은 규모의 프로젝트 개발에 유리하다.

Graph-QL과 REST-API

: 웹 개발에서 사용되는 API 설계 스타일


Graph-QL

: FaceBook에서 개발된 클라이언트가 필요로하는 데이터를 정확하게 요청할 수 있도록 하는 효율적인 데이터 쿼리 언어이다. 단일 엔드포인트를 통해 데이터를 요청, 반환하며 클라이언트의 요청에 따라 유연하게 데이터를 제공하므로, 과도하거나 불필요한 데이터를 가져오는 문제를 해결할 수 있다. 또, Graph-QL은 타입 시스템을 사용하여 데이터 모델을 정의하고, 클라이언트에게 사용가능한 데이터를 명확하게 알려준다.


REST-API

: 웹 애플리케이션의 자원을 나타내고 상호작용하는 방식을 기반으로 한 아키텍처 스타일이다. REST-API는 URL(Uniform Resourse Locator)로 자원을 표현하고, HTTP 메서드를 사용하여 자원을 조작한다. 클라이언트와 서버 간의 통신을 상태없이(stateless)처리하며, 서버는 클라이언트의 요청을 이해하고 응답을 제공한다. 자원을 여러 개의 엔드포인드로 나누어 표현하고, 각 엔드포인트는 특정한 데이터를 반환한다. 클라이언트는 필요한 데이터를 요청할 때마다 새로운 요청을 보내야 한다.

UI/UX

: UI(User Interface)는 사용자가 애플리케이션 또는 웹 사이트와 상호작용할 때 직접적으로 보이고 사용하는 모든 요소를 의미한다. 즉, 버튼, 입력 필드, 메뉴, 아이콘, 레이아웃 등과 같은 시각적 요소와 사용자와 상호작용하는 기능을 담당하는 컨트롤을 의미한다. UI는 사용자가 쉽게 이해하고 사용할 수 있도록 디자인 되어야하며, 좋은 UI는 일관성, 직관성, 가독성 등을 갖추어 사용자의 경험을 향상시킨다.
UX(User Experience)는 사용자가 제품 또는 서비스를 사용하는 동안 느끼는 전반적인 경험을 의미한다. 즉, 사용자가 애플리케이션 또는 웹 사이트를 사용할 때 얻는 만족도, 편의성, 효율성을 포함한다. UX는 UI뿐만 아니라 정보 구조, 내비게이션, 콘텐츠 가치, 상호작용 흐름 등을 고려하여 사용자가 원활하게 목표를 달성할 수 있는 경험을 제공해야 한다. 좋은 UX는 쉬운 접근성, 직관적인 인터페이스, 효율적인 작업 흐름 등을 제공한다.
UI는 UX의 일부이며, UI/UX는 디자인과 개발 과정에서 긴밀하게 협력하여 사용자에게 효과적인 디지털 경험을 제공한다.

스벨트

: 스벨트(Svelte)는 웹 프론트엔드 개발을 위한 자바스크립트 프레임워크이다. 컴파일러를 사용하여 빌드 시점에서 코드를 변환하고, 실행 시에는 가벼운 런타임을 사용하여 최적화된 웹 애플리케이션을 생성한다. 또한 스벨트는 적은 양의 코드로 작성하고, 가상 돔을 사용하지 않으며 진짜 반응성(Reactive)를 제공한다.

스벨트(Svelte) 특징 정리
  • 선언적인 구성 : HTML, CSS, JavaScript를 기반으로 구성, 각각의 코드를 선언적으로 작성한다. -> 더 직관적이고 가독성 높은 코드를 작성할 수 있다.
  • 리액티브 선언문 : 컴포넌트의 상태 변화를 추적하고, 자동으로 화면을 업데이트하는 리액티브 선언문을 제공한다. -> 상태 변화에 대한 명시적인 코드를 작성하지 않아도 된다.
  • 컴파일러 기반 접근 : 컴파일러를 사용하여 컴포넌트를 가상 DOM 대신 실제 DOM으로 변환한다. -> 초기 로딩 속도 향상 및 메모리 사용량 축소로 더 가벼운 애플리케이션을 구현할 수 있다.
  • 코드 최적화 : 불필요한 코드를 제거하고, 변화가 필요한 부분만을 업데이트하는 방식으로 작동한다. -> 성능을 향상시키고, 애플리케이션의 용량을 줄일 수 있다.
  • 컴포넌트 기반 아키텍처 : 컴포넌트 기반 아키텍처를 채택하여 재사용 가능한 UI요소를 작성할 수 있게 한다. -> 코드의 재사용성과 유지보수성을 향상시킬 수 있다.

CI/CD

: 개발자들이 소프트웨어를 효율적으로 빌드, 테스트, 배포할 수 있도록 지원하는 개발 방법론 및 프로세스이다. CI(Continuous Integration)은 개발자들이 개별적으로 작업한 코드를 정기적으로 통합하는 프로세스를 의미하는 지속적인 통합을 뜻하고, CD(Continuous Delivery)는 개발된 소프트웨어가 자동으로 빌드, 테스트, 승인 과정을 거쳐 배포되는 것을 의미하는 지속적인 배포를 뜻한다. CI/CD를 통해 버그를 조기에 발견, 해결할 수 있고, 새로운 기능을 신속하게 사용자에게 제공할 수 있으며, 코드 충돌을 최소화할 수 있다. 또한 CI/CD는 개발자들이 더 빠른 피드백과 안정적인 개발 환경을 경험할 수 있고, 개발자와 운영팀 간 협업을 강화시킨다.
