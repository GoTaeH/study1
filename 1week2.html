애자일 기법

: 애자일(Agile)기법은 소프트웨어 개발을 위한 프로젝트 관리 및 개발 방법론 중 하나로, 변화에 유연하게 대응하고 빠른 배포를 지향하는 접근 방식이다. 개발 과정에서 요구사항이나 우선순위를 유연하게 조정하며 작은 주기로 반복적인 개발을 수행하여 효율적인 개발과 신속한 시장 반응으로 가능하게 한다. 애자일 기법을 따르는 대표적인 예시는 Scrum, XP(eXtreme Programming), Kanban 등이 있다.

애자일 기법 주요 특징
  • 개인과 개인 간의 상호작용이 프로세스 및 툴보다 우선
  • 작동하는 소프트웨어가 포괄적인 문서보다 우선
  • 고객과의 협업이 계약 협상보다 우선
  • 변화에 대응하는 것이 계획을 따르는 것보다 우선

애자일 기법 장점/단점

장점 : 개발 과정이 빠르고 유연, 짧고 반복적인 스프린트로 구성되어 빠른 결함 식별 및 수정 가능, 소규모 팀들이 여러과제 각각 할당받아 처리 가능, 개발과정 중 신속하게 제품 변경 가능
단점 : 빠른 반복 작업에 익숙한 숙련된 사람 필요, 수많은 변경사항 있을 수 있어 번거로움 발생 가능

폭포수 기법

: 폭포수(Waterfall)기법은 소프트웨어 개발 프로세스의 전통적인 접근 방법 중 하나이다. 단계적인 진행 방식으로, 각 단계가 순차적으로 이루어지며 이전 단계가 완료된 후에만 다음 단계로 진행한다.

폭포수 기법 주요 특징
  • 단계적 진행 방식 : 요구사항 정의, 시스템 설계, 개발, 테스트, 배포 등의 단계를 순차적으로 진행
  • 강력한 계획과 문서화 : 프로젝트 시작 시 요구사항을 명세화하고, 강력한 계획과 문서화를 통해 프로젝트 진행을 관리
  • 엄격한 제어와 변경 관리 : 변경에 대한 엄격한 제어를 가져 변경이 발생하면 해당 단계로 돌아가거나 추가 비용, 시간이 소요
  • 위험 분석과 테스트 : 프로젝트 초기에 위험 요소 분석, 개발 단계에서 테스트를 진행하여 품질 보장에 노력

폭포수 기법 장점/단점

장점 : 팀 규모에 상관없이 따르기 쉬움, 새로운 프로젝트 안정적으로 시작 가능, 실행하기 수월하며 목표를 자주 변경하지 않을 수 있음, 예상결과와 리스크 통제하기 쉬움
단점 : 개발속도가 느리고 유연성 떨어짐, 변경이 자유롭지 못함

익스트림 기법

: 익스트림(eXtreme Programming, XP)기법은 애자일 방법론 중 하나로, 고품질의 소프트웨어를 신속하게 개발하기 위해 간결한 기술과 프로세스를 강조하는 접근 방식이다. 릴리즈 기간을 짧게 반복하면서 고객의 요구사항 반영에 대한 가시성을 높이고, 릴리즈 테스트마다 고객을 직접 참여시킴으로 요구한 기능이 제대로 구현되는지 고객이 직접 확인할 수 있다.

익스트림 기법 핵심 가치 5가지

: 의사소통, 단순성, 피드백, 용기, 존중


익스트림 기법 실천 방법
  • 짝 프로그래밍(Pair Programming) : 두 명의 개발자가 한 컴퓨터에서 협력하여 코드를 작성, 지식 공유, 실수 감소, 코드 품질 향상 목적
  • 테스트 주도 개발(Test-Driven Development) : 실제 코드를 작성하기 전에 테스트 케이스를 먼저 작성
  • 전체 팀(Whole Team) : 개발에 참여하는 모든 구성원은 각자의 역할이 있고 책임이 필요
  • 계속적인 통합(Continuos Integration) : 모듈 단위로 나눠서 개발된 코드들은 하나의 작업이 마무리될 때마다 지속적으로 통합
  • 디자인 개선(Design Improvement) : 프로그램 기능의 변경 없이 단순화, 유연성 강화 등을 통해 시스템 재구성
  • 소규모 릴리즈(Small Release) : 릴리즈 기간을 짧게 반복함으로써 고객의 요구 변화에 신속 대응 가능

리덕스

: 리덕스(Redux)는 자바스크립트 애플리케이션을 위한 예측 가능한 상태 컨테이너이다. 애플리케이션의 상태를 중앙 집중식으로 관리하여 상태 변화를 예측 가능하고 효율적으로 관리할 수 있다.
리덕스는 주로 React와 함께 사용되며, React 컴포넌트와 상태를 연결하여 애플리케이션 전반에서 일관된 상태 관리를 제공한다.

리덕스 주요 개념
    1. 상태(State) : 리덕스는 애플리케이션의 전역 상태를 객체 형태로 관리한다. 상태는 읽기 전용이며 직접 변경 불가능하고 애플리케이션의 데이터 모델이며 컴포넌트 간에 공유된다.
    2. 액션(Action) : 상태에 대한 변화를 나타내는 객체이다. type 필드를 가져야 하며, 필요에 따라 추가적인 데이터를 포함할 수 있고, 디스패치(dispatch) 되어야 한다.
    3. 리듀서(Reducer) : 현재 상태와 액션을 받아 새로운 상태를 반환하는 순수 함수이다. 이전 상태를 변경하지 않고, 새로운 상태 객체를 생성하여 반환해야 한다.
    4. 스토어(Store) : 상태, 액션, 리듀서를 관리하는 리덕스의 핵심 객체이다. 중앙 집중식으로 상태를 관리하고 상태 변화를 구독할 수 있는 기능을 제공한다.
    5. 디스패치(Dispatch) : 액션을 스토어에 전달하여 상태 변화를 일으킨다.

클라우드 서버

: 클라우드 서버(Cloud Server)는 인터넷을 통해 접근할 수 있는 가상화된 컴퓨터 리소스이다. 기존의 물리적인 서버와는 다르게 클라우드 컴퓨팅 환경에서 제공되며 필요에 따라 유연하게 스케일링할 수 있다. 전력, 위치, 서버 세팅, 확장성을 고민하지 않고 서비스 운영에만 집중할 수 있다는 장점이 있다.

클라우드 서버 주요 특징
  • 가상화 : 물리적인 서버를 여러개의 가상 서버로 분할
  • 확장성 : 필요에 따라 자원을 동적으로 확장, 축소 가능 -> 최적의 성능 유지 가능
  • 유연성 : 사용자가 필요로 하는 운영체제와 소프트웨어 환경 선택 가능
  • 접근성 : 인터넷을 통해 어디서나 접근 가능
  • 유지보수와 관리 : 클라우드 서비스 제공 업체가 관리하여 사용자는 서버의 유지보수, 보안 패치, 하드웨어 업그레이드 등과 같은 작업을 직접 처리할 필요 없음

NGINX

: 트래픽이 많은 웹사이트의 서버(WAS)를 도와주는 비동기 이벤트 기반구조의 경량화 웹 서버 프로그램이다. 정적 파일을 응답해주는 HTTP Web Server로 활용되기도 하고, Reverse Proxy Server로 활용하여 WAS의 부하를 줄일 수 있는 로드 밸런서 역할을 하기도 한다.

NGINX 장점
  • 뛰어난 성능 : 비동기 이벤트 기반 아키텍처를 사용해 높은 처리량과 낮은 대기 시간을 제공
  • 확장성 : 동시 연결 처리 능력과 로드 밸런싱 기능을 통해 서비스 확장이 용이
  • 리버스 프록시 : 백엔드 서버의 보안, 부하 분산, 캐싱 등의 기능 제공
  • SSL/TLS 암호화 : 웹 트래픽의 암호화를 지원하여 보안 통신 제공
  • 정적 파일 서빙 : 정적파일을 효율적으로 제공해 성능 향상
  • 가상 호스팅 : 하나의 서버에서 여러 개의 도메인 호스팅 가능 -> 비용, 리소스 절약 가능

NGINX 단점
  • 동적 콘텐츠 처리 : 주로 정적 파일 서빙에 특화되어 동적 콘텐츠 처리에는 제한이 있기에 추가적인 설정이 필요
  • 복잡한 설정 : 설정 파일은 유연성을 제공하지만, 초보자에게는 다소 복잡할 수 있어 학습이 필요
  • 커스터마이징의 어려움 : C언어로 개발되었으며, 사용자 정의 모듈이나 기능 추가를 위해서는 컴파일 필요 -> 커스터마이징이 어렵거나 번거로울 수 있음

일렉트론

: 노드JS를 기반으로 자바스크립트, HTML, CSS를 사용하여 데스크톱 애플리케이션을 만드는 오픈소스 프레임워크이다. 일렉트론(Electron)은 오픈소스 웹브라우저 크로미움(Chromium), 브라우저 밖에서 자바스크립트 코드를 실행하는 런타임 환경인 노드JS, 오픈소스 자바스크립트 엔진 V8로 구성되어 있다.

일렉트론 장점
  • 낮은 진입 장벽 : 새로운 기술이나 언어를 배울 필요가 없어 개발자의 진입 장벽이 낮음
  • 개발 속도 향상 : 웹 애플리케이션의 비즈니스 로직, 디자인 및 구조를 재사용해 개발에 소요되는 시간과 비용 축소
  • 크로스 플랫폼 지원 : 한번에 윈도우, 맥OS, 리눅스에 대한 빌드를 지원, 설치 가능하도록 할 수 있음 -> 기존의 웹 기술 그대로 사용하면서 브라우저 버전별로 기능을 제약하거나 타겟 플랫폼에 따라 별도의 버전 유지할 필요 없음
  • 풍부한 OS 네이티브 API를 가진 런타임 제공 : 자바스크립트로 OS의 시스템 피처들을 손쉽게 사용 가능
  • 써드파티 지원 : 노드JS의 모든 빌트인 모듈과 써드파티 모듈 사용이 가능해 네트워크 리소스를 제약없이 사용할 수 있음
  • 노드 접근 권한 관리 : 웹뷰에서 노드JS에 대한 접근 권한을 관리 할 수 있음
  • 빌드 툴과 인스톨러 제공 : 일렉트론 데스크톱 애플리케이션을 손쉽게 패키징할 수 있고, 인스톨러도 순식간에 만들 수 있음.

일렉트론 단점
  • 큰 설치 파일 용량
  • 상대적으로 느린 속도
  • 보안은 개발자의 책임
  • 크로스 플랫폼 빌드

Dev-ops

: 소프트웨어 개발과 IT 운영을 통합하는 문화, 철학, 방법론의 조합을 의미한다. Dev는 개발(Development)을, Ops는 운영(Operations)을 나타내며, 두 영역 간의 협력, 커뮤니케이션, 자동화를 강조한다. 이로 인해 애플리케이션의 품질과 안정성을 향상시키며 더 빠른 속도로 소프트웨어를 제공하는 데 도움을 준다.

플러터

: 플러터(Flutter)는 구글에서 개발한 오픈 소스 UI 프레임워크로 크로스 플랫폼 모바일 애플리케이션 개발을 위해 사용된다. Dart 언어로 작성되며, 단일 코드베이스로 안드로이드와 iOS용 애플리케이션을 동시에 개발할 수 있다.

플러터 주요 특징
  • 높은 성능 : 네이티브 컴파일된 코드로 실행되어 뛰어난 성능을 제공한다.
  • 반응형 UI : 자체적인 위젯을 사용하고, 풍부한 애니메이션 라이브러리 및 확장 가능한 계층형 아키텍처를 제공함으로 다이나믹하고 멋진 UI를 구성할 수 있다.
  • 핫 리로드(Hot Reload) : 핫 리로드 기능을 제공하여 코드 수정 후에도 애플리케이션을 즉시 반영할 수 있다. 이로 인해 빠른 실험과 디버깅이 가능해지고, 생산성을 높일 수 있다.

리액트 네이티브(엑스포)

: 리액트 네이티브(React Native)는 페이스북에서 개발한 오픈 소스 모바일 애플리케이션 프레임워크이다. 자바스크립트와 리액트를 기반으로 안드로이드 및 iOS 플랫폼용 네이티브 애플리케이션을 개발할 수 있다.
리액트 네이티브는 자바스크립트 코드를 사용하여 UI를 빌드하고, 플랫폼 API를 제공한다. 이를 통해 개발자는 하나의 코드베이스로 여러 플랫폼에 대한 애플리케이션을 개발할 수 있다.

리액트 네이티브 사용 방식 2가지
  • 순수 리액트 네이티브 사용
  • 엑스포(Expo)라는 툴킷 사용 (엑스포: 리액트 네이티브 애플리케이션 개발을 위한 추가적인 개발 도구와 서비스를 제공하는 플랫폼)

리액트 네이티브 주요 특징
  • 크로스 플랫폼 개발 : 단일 코드베이스로 개발할 수 있어 개발 생산성을 향상시키고 개발 비용을 절감할 수 있다.
  • 리액트 기반 : 리액트의 개념과 구문을 사용하여 UI를 구축하기 때문에 컴포넌트 기반 접근법과 가상 DOM(Virtual DOM)을 활용할 수 있다.
  • 네이티브 성능 : 네이티브 컴포넌트와 상호작용하여 네이티브 애플리케이션과 유사한 성능을 제공한다.
  • 핫 리로딩 : 핫 리로딩(Hot Reloading) 기능을 제공하여 코드 변경 사항을 실시간으로 반영할 수 있다.
  • 엑스포 플랫폼 : 네이티브 개발 환경을 설정하거나 빌드 설정을 수동으로 처리하지 않고도 애플리케이션 개발에 집중할 수 있다.
