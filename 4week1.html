블록태그와 인라인태그

블록태그

: 웹 문서의 구조를 정의하고, 콘텐츠를 블록 단위로 배치하는 데 사용된다. 블록 요소는 기본적으로 가로 폭의 100%를 차지하며, 새로운 줄에서 시작한다. 블록 요소는 다른 블록 요소나 인라인 요소를 포함할 수 있다.

블록태그 종류

: <div>, <address>, <article>, <aside>, <audio>, <blockquote>, <canvas>, <dd>, <dl>, <fieldset>, <figcaption>, <figure>, <footer>, <form>, <h1>, <h2>, <h3>, <h4>, <h5>, <h6>, <header>, <hgroup>, <hr>, <noscript>, <ol>, <output>, <p>, <pre>, <section>, <table>, <ul>, <video>

인라인태그

: 텍스트나 이미지와 같은 콘텐츠를 블록 단위로 구분하지 않고, 문장 내에 삽입하는 데 사용된다. 인라인 요소는 콘텐츠가 끝나는 지점까지만 차지하며 새로운 줄에서 시작하지 않는다.

인라인태그 종류

: <span>, <a>, <abbr>, <acronym>, <b>, <bdo>, <big>, <br/>, <button>, <cite>, <code>, <dfn>, <em>, <i>, <img>, <input>, <kbd>, <label>, <map>, <object>, <q>, <samp>, <small>, <script>, <select>, <strong>, <sub>, <sup>, <textarea>, <tt>, <var>

태그 중첩 규칙
  • 블록태그는 자식으로 또다른 블록태그나 인라인태그를 가질 수 있다.
  • 인라인태그는 자식으로 또다른 인라인태그만 올 수 있다.
  • 블록태그 중 <p>태그는 자식으로 인라인 태그만 가질 수 있다.

반응형 웹

: 반응형 웹(Responsive Web)은 다양한 디바이스와 화면 크기에 맞게 웹 사이트가 자동으로 최적화되는 웹 디자인 기법을 말한다. 즉, 사용자가 데스크톱 컴퓨터, 노트북, 태블릿, 스마트폰과 같은 다양한 기기로 웹 사이트를 접속해도 모든 기기에서 적절한 레이아웃과 디자인으로 제공되어 사용자 경험을 향상시키는 방식이다. 반응형 웹은 CSS의 미디어 쿼리를 사용하여 기기의 화면 크기와 특성을 감지한다.

반응형 웹 구현 기술
  • 미디어 쿼리 : CSS의 미디어 쿼리를 사용하여 화면 크기와 기기의 특성을 감지, 이에 따라 스타일을 적용한다.
  • 유동형 그리드 : 상대적인 단위인 %를 사용하여 그리드 시스템을 구성하여 화면 크기에 따라 레이아웃이 유동적으로 변하도록 한다.
  • 유동형 이미지 : 이미지의 크기를 상대적인 크기로 지정하여 화면에 맞게 조정될 수 있도록 한다.
  • 뷰포트 : 모바일 기기에서 화면에 표시되는 영역을 조절하는 메타 태그를 사용하여 뷰포트를 설정한다.

CSS 전처리기 / 후처리기

CSS 전처리기

: 웹 개발자들이 CSS 작성을 간편하게 만들기 위해 사용되는 언어로 변수, 중첩 규칙, 함수, 연산자 등과 같은 기능을 추가하여 CSS 코드를 확장한다. 기능들은 CSS 컴파일러를 통해 일반적인 CSS로 변환되어 웹 페이지에서 사용된다.
전처리기 : Scss, Less, Stylus


전처리기 장단점

장점 :
재사용성 - 공통 요소 또는 반복적인 항목을 변수, 함수로 대체 가능
시간적 비용 감소 - 임의 함수 및 내장 함수로 인한 개발 시간적 비용 절약
유지 관리 - CSS 코드를 여러 파일로 나눠 유지 보수성 향상
Learning curve 낮음
단점 : 전처리기를 위한 도구 필요, 퍼블리셔나 디자이너가 개발에 대한 역할과 요소 접목으로 개발적인 요소 알아야 함

CSS 후처리기

: 일반적인 CSS를 더욱 강력하고 효율적으로 만들기위해 사용되는 도구로, 일반적인 자동화, 코드 압축, 벤더 프리픽스 추가 등과 같은 작업을 수행한다. 후처리기는 CSS를 기반으로 하며, 웹 개발 프로젝트를 빌드할 때 주로 사용한다.
후처리기 : PostCSS, Autoprefixer


모나드

: 모나드(Monad)는 함수형 프로그래밍에서 사용되는 개념으로, side effect(부작용)를 추상화하고 조합하기 위한 패턴이다. 일반적으로 순수 함수형 언어에서 부작용을 처리하는 방법을 제공하며, 안전하고 예측 가능한 코드 작성을 돕는다.

모나드 주요 기능
  • 컨텍스트 : 값들의 컨텍스트를 가지고 있는데, 이는 값의 부작용, 비동기 작업, 예외 처리 등과 관련된 정보를 포함하고 있다.
  • flatMap / bind : 컨텍스트 내의 값을 꺼내서 변환하는 함수를 제공한다. 보통 flatMap 또는 bind 라고 불린다.
  • unit / return : 값을 컨텍스트에 넣는 함수를 제공한다. 보통 unit 또는 return 이라고 불린다.
